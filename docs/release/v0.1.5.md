# iqdb-filter v0.1.5 — Scaffold Finalization

**Tightening the bootstrap.** v0.1.5 is the last step before domain logic: still no evaluator, just the housekeeping that pins the scaffold to the iQDB standard — co-authorship and a consistent MSRV — so v0.2.0 lands on a clean, correct foundation.

## What is iqdb-filter?

The metadata-filtering layer of the iQDB vector-database spine — the one place that decides what a `Filter` means. Every index that honours metadata filters (`iqdb-flat`, and later `iqdb-hnsw` / `iqdb-ivf`) delegates here, so the semantics can never drift between implementations. It evaluates the `Filter` expression language defined in `iqdb-types` and sits one level above it.

## What's new in 0.1.5

- **Authors** — added co-author **Matt Callahan** to the crate metadata.
- **MSRV** — pinned to Rust `1.87` consistently across `Cargo.toml`, `clippy.toml`, the README badge, `dev/DIRECTIVES.md`, and the CI matrix (previously `1.85` in some places). One MSRV, stated once, enforced everywhere.

No public API exists yet; there is nothing to break.

## Breaking changes

**None** — there is no domain surface in the scaffold.

## Verification

The scaffold compiles clean and the placeholder smoke test passes across the CI matrix:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
```

MSRV: Rust 1.87.

## What's next

- **v0.2.0 — the canonical filter evaluator.** The hard part, not deferred: `FilterEvaluator` with validation-on-construction and an infallible per-row boolean evaluator, with the closed-world null/absent-field semantics pinned by property and conformance tests.
