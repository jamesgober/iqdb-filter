# iqdb-filter v1.0.0 — Stable

**The filter layer is committed.** Five 0.x releases built and froze the surface; v1.0.0 puts it under the SemVer 1.x guarantee — no breaking changes until 2.0. There is no new API here. This is the stability promise: what you build against today keeps compiling.

## What is iqdb-filter?

The metadata-filtering layer of the iQDB vector database. It owns one definition of what a `Filter` means — so every index that honours filters produces identical results — and the machinery to apply it efficiently: scan helpers, a selectivity-driven strategy selector, and an opt-in inverted index.

## The 1.0 surface

```rust
use iqdb_filter::{FilterEvaluator, MetadataIndex, choose_strategy};
use iqdb_types::{Filter, Metadata, Value};

// 1. Validate once.
let evaluator = FilterEvaluator::new(Filter::and(vec![
    Filter::eq("lang", Value::String("rust".into())),
    Filter::gt("year", Value::Int(2020)),
]))
.expect("valid filter");

// 2. Index the selective field.
let rows = [
    (0_usize, [("lang".to_string(), Value::String("rust".into())), ("year".to_string(), Value::Int(2026))].into_iter().collect::<Metadata>()),
    (1, [("lang".to_string(), Value::String("go".into())), ("year".to_string(), Value::Int(2024))].into_iter().collect::<Metadata>()),
];
let index = MetadataIndex::build(&["lang"], rows.iter().map(|(k, m)| (*k, Some(m))));

// 3. Narrow via the index (superset), confirm with `evaluate`.
let hits: Vec<usize> = match index.candidates(&evaluator) {
    Some(cands) => cands.into_iter().filter(|&k| evaluator.evaluate(rows.iter().find(|(j, _)| *j == k).map(|(_, m)| m))).collect(),
    None => rows.iter().filter(|(_, m)| evaluator.evaluate(Some(m))).map(|(k, _)| *k).collect(),
};
assert_eq!(hits, [0]);

// And let the crate pick the strategy.
let _ = choose_strategy(&evaluator);
```

The committed surface (frozen, recorded in `dev/ROADMAP.md`):

- **`FilterEvaluator`** — `new`, `evaluate`, `filter`, `prefilter`, `postfilter`.
- **`MetadataIndex<K>`** — `build`, `candidates`, `estimate_selectivity`, `len`, `is_empty`, `is_indexed`, `indexed_fields`.
- **Selection** — `estimate_selectivity`, `choose_strategy`, `StrategySelector` (`new` / `with_prefilter_threshold` / `prefilter_threshold` / `choose` / `choose_with_index`), `FilterStrategy`.
- **Consts** — `MAX_FILTER_DEPTH`, `MAX_IN_VALUES`, `DEFAULT_PREFILTER_THRESHOLD`, `VERSION`.

## Why this is 1.0

Every Definition-of-Done criterion (`dev/DIRECTIVES.md` §7) is met:

- **Clean everywhere** — `fmt`, `clippy -D warnings` (all features and none), `test --all-features`, and `cargo doc -D warnings` pass on the CI matrix (Linux, macOS, Windows), on stable and the 1.87 MSRV.
- **Secure** — `cargo audit` + `cargo deny` clean; **zero `unsafe`** (`#![forbid(unsafe_code)]`); no `unwrap` / `expect` / `todo!` / `dbg!` in shipping code.
- **Correct** — closed-world semantics and the `MetadataIndex` superset contract are pinned by property and conformance tests, and **fuzzed** over unbounded input (the `robustness` and `superset` targets each cleared >250k executions without a crash).
- **Proven sufficient** — the consumer-simulation suite builds a filtered top-`k` searcher on the public API alone and asserts the index-accelerated path equals a full scan for every filter shape.
- **Documented** — every public item carries a runnable rustdoc example; the README leads with the Tier-1 surface; `docs/API.md` covers the whole API; four runnable examples ship in `examples/`.
- **Benchmarked** — the hot paths (`evaluate`, `prefilter`, `candidates`, selectivity) carry `criterion` benches.

`loom` is **not applicable**: `iqdb-filter` has no shared-state or lock-free paths. `FilterEvaluator` and `MetadataIndex` are immutable after construction and `Sync` by composition, so there is nothing for a concurrency model checker to explore.

## Breaking changes

**None.** The public API is unchanged from v0.5.0; this release is the stability commitment.

## What's next (post-1.0, additive)

- **`InFilter` pushdown** into graph traversal — pruning HNSW/IVF branches that can't yield a surviving candidate. It needs a real approximate-index consumer to validate recall, and being additive (`FilterStrategy` is `#[non_exhaustive]`) it will ship in a 1.x MINOR without breaking anything.
